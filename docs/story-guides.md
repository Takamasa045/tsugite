# 王道の物語構成・映像文法

`knowledge/story-frameworks/catalog.yaml` は、企画時に使う34種の構成法、尺別配分、35種のカット設計・連続性・編集・音・AI動画原則を機械可読で保持します。

これは「型に当てはめて自動生成する機能」ではありません。Planner が目的、尺、媒体、視聴者心理から候補を比較し、Gate 1 前の提案を説明可能にする読み取り専用の助言です。

## エージェントから参照する

カタログ全体:

```sh
node bin/pipeline story-guides --json
```

企画に合わせた推薦:

```sh
node bin/pipeline story-guides \
  --request "90秒の物語プロモ。主人公の迷い、選択、成長を映画的に描く" \
  --duration 90 \
  --json
```

推薦結果には次が含まれます。

- 第一候補と選定理由
- 最大2件の補助候補
- 今回は使わない型と不採用理由
- 尺別の推奨カット数と役割配分
- 1カット1役割、開始/終了状態、仕込みと回収
- 180度線、視線、動機のあるカット、J/Lカットなどの映像文法
- 出典と安全上の注意

## 収録する構成法

- 起承転結、三幕構成、一般化ビートシート、ヒーローズジャーニー抽象型
- フライターク、7ポイント、ストーリーサークル、序破急、五幕
- AIDA、PAS、4P、問題解決、Hook Value Proof CTA、PREP、Why How What
- Before After Bridge、ケーススタディ、チュートリアル、ダイジェスト
- 逆三角形、誤解・事実・再定義、比較・判断、証言・実証
- オープンループ、サスペンス、ミステリー、コメディ三段型
- 人物ドキュメンタリー、インタビュー洞察型
- パフォーマンスMV、物語・コンセプトMV、連想モンタージュ、ループ短尺

Save the Cat は固有の15ビートをそのまま短尺へ押し込まず、役割を抽象化した「一般化ビートシート」として扱います。必要なときは `derived_from` と出典から元の方法を確認できます。

## 選定ルール

1. 目的で主軸を選ぶ。
2. 尺で圧縮度を決める。
3. 媒体と視聴者の次の行動で補助線を選ぶ。
4. 第一候補だけでなく、不採用理由も確認する。
5. 採用した型をカットの開始状態、変化、終了状態へ落とす。

映像原則は毎回すべて返しません。第一候補に応じて優先セットを選びます。サスペンスでは保持・無音・視線、ドキュメンタリーでは空間提示・カットアウェイ・ルームトーン、MVでは色・編集リズム・音のモチーフ、コメディでは仕込み・間・速度差が優先されます。共通して、AI生成用の連続性アンカー、編集ハンドル、カメラ動機、1クリップ1主動作を含めます。

構成カタログは実行能力を意味せず、prompt や `project.yaml` を自動変更しません。生成モデル固有の書き方は `guides`、実行可否は adapter / backend と `doctor`、実行承認は各 Gate がそれぞれの truth です。
